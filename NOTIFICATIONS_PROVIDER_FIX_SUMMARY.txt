  Providers need to be registered at the app level in MultiProvider so that:
  1. Provider is created when app starts
  2. Provider is available to entire app
  3. Any screen can access it using context.read() or context.watch()

What was happening before:
  
  1. App starts with MultiProvider (without NotificationsProvider)
  2. Home screen opens - works fine
  3. Tap bell icon â†’ Navigate to NotificationsCenterScreen
  4. NotificationsCenterScreen.initState() tries: 
     context.read<NotificationsProvider>()
  5. ERROR! Provider not found in widget tree
  6. Crash with ProviderNotFoundException

What happens now:
  
  1. App starts with MultiProvider (WITH NotificationsProvider)
  2. NotificationsProvider is created and registered
  3. Provider available to entire app
  4. Home screen opens - works fine
  5. Tap bell icon â†’ Navigate to NotificationsCenterScreen
  6. NotificationsCenterScreen.initState() tries:
     context.read<NotificationsProvider>()
  7. SUCCESS! Provider found in widget tree
  8. fetchNotifications() called successfully
  9. Notifications display normally
